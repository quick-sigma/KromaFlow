# S007 — Abstract Processor and OutputFormatter Classes

## Resumen

Se refactorizó el pipeline de procesamiento de imágenes para introducir dos clases abstractas base:

- **`Processor`** — abstracción para transformar una imagen (crop, resize, rotate, flip, grayscale).
- **`OutputFormatter`** — abstracción para codificar una imagen procesada a bytes (con manejo de compatibilidad de formatos).

Las implementaciones concretas existentes (`process_order`) se mantienen como wrapper de conveniencia, pero ahora delegan internamente en estas abstracciones.

## Detalle

### Branch
- `feature/abstract-processor-formatter` — basado en `master`.

### Clases abstractas — `Backend/base.py`

```python
class Processor(ABC):
    @abstractmethod
    def process(self, image: Image.Image, instructions: ProcessingInstructions) -> Image.Image:
        ...

class OutputFormatter(ABC):
    @abstractmethod
    def format_output(self, image: Image.Image, output_format: str, quality: int | None = None) -> tuple[bytes, str]:
        ...
```

- Usan `abc.ABC` y `@abstractmethod` de la biblioteca estándar.
- No se pueden instanciar directamente.
- Definen el contrato para cualquier implementación concreta.

### Implementaciones concretas

#### `ImageProcessor(Processor)` — `Backend/processor.py`

Método `process()` aplica en orden:
1. Crop
2. Resize
3. Rotate
4. Flip
5. Grayscale

Siempre copia la imagen antes de mutarla. Las funciones `_apply_*` siguen siendo internas reutilizables.

#### `ImageOutputFormatter(OutputFormatter)` — `Backend/output_formatter.py`

Método `format_output()`:
1. Resuelve alias (`jpg` → `jpeg`, `tif` → `tiff`)
2. Valida formato soportado
3. Compatibilidad: RGBA → RGB (fondo blanco) para JPEG
4. Codifica con calidad configurable para formatos lossy

Exporta constantes:
- `SUPPORTED_FORMATS` — conjunto de formatos válidos
- `_FORMAT_ALIASES` — alias de extensiones
- `_CONTENT_TYPE` — mapa formato → MIME type

### `process_order()` refactorizado

Ahora es un wrapper de conveniencia:

```python
def process_order(order: Order) -> tuple[bytes, str]:
    processor = ImageProcessor()
    formatter = ImageOutputFormatter()
    processed = processor.process(order.image, order.instructions)
    return formatter.format_output(processed, order.output_format, order.instructions.quality)
```

Todos los tests existentes siguen pasando sin cambios.

### `main.py` actualizado

Usa instancias a nivel de módulo:

```python
_processor: Processor = ImageProcessor()
_formatter: OutputFormatter = ImageOutputFormatter()
```

El endpoint ahora llama directamente a los métodos abstractos en lugar de a `process_order()`:

```python
async with _processing_lock:
    processed_image = _processor.process(order.image, order.instructions)
    data, content_type = _formatter.format_output(...)
```

Esto permite intercambiar implementaciones (inyección de dependencias).
