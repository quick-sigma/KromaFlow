# S003 — Primer paso: FileInput + i18n

## Resumen
Se creó el branch `step-1-file-processor`, se implementó el componente `FileInput` con soporte multiformato de imágenes siguiendo TDD, y se inicializó i18n con inglés/español.

## Detalle

### Branch
- Se creó el commit inicial del repositorio y el branch `step-1-file-processor` para trabajar el primer paso del procesador de archivos.

### Internacionalización (`i18n/`)
- Dependencias instaladas: `i18next`, `react-i18next`
- Archivos creados:
  - `src/i18n/index.ts` — Configuración de i18next con `initReactI18next`, recursos `en` y `es`, `lng` por defecto `'en'`
  - `src/i18n/locales/en.json` — Traducción: `"fileInput.loadImages": "Load Images"`
  - `src/i18n/locales/es.json` — Traducción: `"fileInput.loadImages": "Cargar Archivos"`
- `src/main.tsx` importa `./i18n` para inicializar al arrancar la app
- `src/setupTests.ts` importa `./i18n` para que esté disponible en los tests

### FileInput — TDD (Test First)

Se escribieron **7 tests** antes de implementar el componente:

1. Renderiza el botón con texto "Load Images" (default EN)
2. Renderiza un elemento `<input>` con `data-testid="file-input"`
3. Acepta formatos: PNG, JPEG, GIF, WebP, AVIF, BMP, TIFF, SVG (verifica `accept` en el input)
4. Permite selección múltiple (`multiple = true`)
5. Muestra "Cargar Archivos" al cambiar idioma a `es`
6. Llama al `onChange` al seleccionar un archivo
7. Forwardea props adicionales al input (e.g. `id`, `className`)

### FileInput — Implementación

- `src/components/FileInput.tsx` — Componente que:
  - Renderiza un `<input type="file">` oculto con `accept` para todos los formatos de imagen
  - Muestra un botón `<button>` estilado con Tailwind (bg-blue-600) que activa el input oculto
  - Usa `useTranslation()` de react-i18next para el texto del botón
  - Props: `onChange` + resto de props de input (via spread)

### App
- `src/App.tsx` se simplificó: eliminó el store de ejemplo (counter) y ahora solo renderiza el título y `<FileInput />`
- `src/App.test.tsx` se actualizó a 2 tests (título + FileInput presente)
