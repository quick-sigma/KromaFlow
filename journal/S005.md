# S005 — MiniatureList + Images Store

## Resumen
Se creó un store Zustand para gestionar las imágenes cargadas y un componente `MiniatureList` que muestra la lista de miniaturas. Se integró `FileInput` con el store para que las imágenes se agreguen automáticamente al cargarlas.

## Detalle

### Branch
- `step-3-miniature-list` — basado en `master` (que ya incluye step 1 y step 2).

### Images Store — `src/stores/images.ts`

Store Zustand que mantiene el estado global de las imágenes cargadas:

| Acción | Descripción |
|---|---|
| `addImages(files: File[])` | Convierte cada `File` a blob URL (`URL.createObjectURL`) y añade una entrada con `id` único (`crypto.randomUUID()`), `name`, `file` y `src`. |
| `removeImage(id: string)` | Elimina una imagen por ID y revoca su blob URL (`URL.revokeObjectURL`). |
| `processImage(id: string)` | Placeholder para procesamiento futuro (actualmente hace un `console.log`). |
| `clearImages()` | Elimina todas las imágenes y revoca todos los blob URLs. |

Tipo `ImageEntry`:
```typescript
type ImageEntry = {
  id: string
  file: File
  src: string   // blob URL
  name: string
}
```

### MiniatureList — `src/components/MiniatureList.tsx`

- Lee `images`, `removeImage`, `processImage` del store Zustand.
- **Estado vacío**: Muestra "No images loaded" (traducido vía i18n).
- **Con imágenes**: Renderiza un `flex flex-wrap` con una instancia de `MiniatureImageWithOptions` por cada entrada.
- Cada `Remove` invoca `removeImage(id)` en el store.
- Cada `Process` invoca `processImage(id)` en el store.

### App.tsx — Integración

- `FileInput.onChange` → extrae `File[]` del evento y llama a `addImages(Array.from(files))`.
- Resetea `event.target.value = ''` para permitir recargar el mismo archivo.
- Renderiza `<MiniatureList />` debajo del FileInput.
- Layout ajustado a `p-8` en lugar de `justify-center` vertical para soportar scroll.
