# S019 — UI redesign: dropzone, dual gallery shelves, processed results display

## Summary

Redesigned the image upload area and gallery layout. The old button-based file picker is now a full drag-and-drop zone with an upload icon. The image list is split into two shelf-like galleries — "To Process" (originals) and "Processed" (results from backend). Processed images are stored and displayed in UI instead of only triggering a download.

## Design

### Dropzone
```
┌──────────────────────────────────────┐
│                                      │
│            ▲ (HiUpload)              │
│     Load or drag images              │
│  PNG, JPEG, GIF, WebP, AVIF...      │
│                                      │
└──────────────────────────────────────┘
```
- Dashed border (gray-600, hover → gray-500)
- `shadow-[inset_0_2px_8px_rgba(0,0,0,0.35)]` for sunken effect
- Hover: deeper shadow + bg tint
- Drag-active: blue border + blue glow inset
- Full-screen overlay when files dragged over document

### Gallery shelves
```
To Process (3) ────────────────────────╌╌╌╌
┌──────────────────────────────────────┐
│  [img] [img] [img]                   │
│                                      │
└──────────────────────────────────────┘

Processed (2) ─────────────────────────╌╌╌╌
┌──────────────────────────────────────┐
│  [img] [img]                         │
│                                      │
└──────────────────────────────────────┘
```
- Each section has a header with gradient line decoration
- Content cards sit in a rounded inset panel (`shadow-[inset_0_2px_6px_rgba(0,0,0,0.25)]`)
- "To Process" shows MiniatureImageWithOptions (with Process + Remove buttons)
- "Processed" shows ProcessedImageCard (with Remove button only, plus image name label)

## Processed image flow

1. User clicks **Process** on an original image
2. Image sent to backend via existing API
3. On success, response blob is:
   - Saved to IndexedDB (`processed-blob-{uuid}`)
   - Added to store's `processedImages` array with metadata
   - Triggered as download (preserving existing behavior)
4. The new processed card appears immediately in the "Processed" gallery
5. On page refresh, both originals and processed images are restored from IndexedDB
