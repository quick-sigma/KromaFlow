# S006 — Image Processing Order Endpoint

## Resumen

Se creó el primer endpoint real del backend (`POST /api/images/process`) que recibe una imagen, la envuelve en una abstracción `Order` con metadatos (recipient, instructions, output_format), la procesa con Pillow según instrucciones estructuradas JSON, y devuelve la imagen procesada directamente.

Se introdujo un lock `asyncio.Lock` para garantizar que los pedidos se procesen uno a la vez (cola interna).

### Models — `Backend/models.py`

```python
@dataclass
class Order:
    image: Image.Image     # Pillow Image (la abstracción central)
    recipient: str         # Usuario que hace el pedido
    instructions: ProcessingInstructions  # Instrucciones estructuradas
    output_format: str     # png, jpeg, webp, gif, bmp, tiff
```

`ProcessingInstructions` (Pydantic `BaseModel`):

| Campo | Tipo | Validación |
|---|---|---|
| `resize` | `ResizeInstruction` (width, height, percent) | width/height > 0; percent 0-1000 |
| `rotate` | int | 0-359 grados |
| `flip` | str | "horizontal" o "vertical" |
| `grayscale` | bool | — |
| `crop` | dict | keys left/top/right/bottom enteros no-negativos |
| `quality` | int | 1-100 (default 85) |

### Procesador — `Backend/processor.py`

Aplica las transformaciones en este orden:
1. **Crop** — recorta usando `Image.crop()`
2. **Resize** — redimensiona con `Image.LANCZOS`
3. **Rotate** — rota con `expand=True`
4. **Flip** — volteo horizontal/vertical (`transpose`)
5. **Grayscale** — convierte a modo `L`
6. **Compatibilidad de formato** — RGBA→RGB para JPEG (fondo blanco)
7. **Codificación** — guarda a bytes con calidad configurable

Soporta alias de extensión: `jpg` → `jpeg`, `tif` → `tiff`.

### Endpoint — `Backend/main.py`

```
POST /api/images/process
Content-Type: multipart/form-data

Fields:
  image: UploadFile (required)
  recipient: str (required)
  instructions: str (required, JSON)
  output_format: str (default "png")
```

Flujo:
1. Parsea el JSON de `instructions` → valida que sea dict
2. Construye `ProcessingInstructions` (Pydantic) → validación estructural
3. Lee el archivo con Pillow (`Image.open` + `load()`)
4. Crea un `Order(image, recipient, instructions, output_format)`
5. Adquiere `_processing_lock` (procesamiento serializado)
6. Llama a `process_order(order)` → recibe `(bytes, content_type)`
7. Devuelve `Response(content=data, media_type=content_type)`

Errores manejados (HTTP 400/422):
- JSON inválido en instructions
- instructions no es un objeto
- Formato de salida no soportado
- Archivo subido no es una imagen válida
- Validaciones de Pydantic fallan (rotate inválido, flip inválido, etc.)

### Seguridad

- El endpoint acepta cualquier UploadFile, pero solo procesa imágenes válidas (Pillow rechaza binarios corruptos/no-imagen).
- No hay límite de tamaño de archivo configurado (depende del deploy/proxy).
- Las instrucciones JSON están validadas por Pydantic (tipos, rangos).
- El `asyncio.Lock` serializa el procesamiento, evitando contención de recursos.
- `Pillow` no ejecuta código del contenido de la imagen (solo decodifica).
