# S008 — WatermarkRemoverProcessor (Gemini Watermark Removal)

## Resumen

Se creó `WatermarkRemoverProcessor`, una implementación concreta de `Processor` que elimina marcas de agua visibles de Gemini (Nano Banana) usando Reverse Alpha Blending. Integra la librería Python `remove-ai-watermarks` (GeminiEngine).

Se agregó el flag `remove_watermark` a `ProcessingInstructions` para que el frontend pueda solicitar la eliminación de watermark como paso previo a las transformaciones.

## Detalle

### Branch
- `feature/watermark-remover` — basado en `master`.

### Librería externa: `remove-ai-watermarks`

| Propiedad | Valor |
|---|---|
| Paquete PyPI | `remove-ai-watermarks==0.12.1` |
| Dependencia adicional | `opencv-python-headless==5.0.0.93` |
| Clase principal | `GeminiEngine` |
| Algoritmo | Reverse Alpha Blending: `original = (wm - α·logo) / (1 - α)` |
| Detección | NCC (Normalized Cross-Correlation) multi-etapa |
| Velocidad | ~0.05s por imagen, CPU only |

### WatermarkRemoverProcessor — `Backend/watermark_remover.py`

Implementa `Processor.process(image, instructions)`:

1. Convierte Pillow Image → numpy array (RGB) → OpenCV compatible
2. Llama a `GeminiEngine.detect_watermark()` con detección automática
3. Si se detecta watermark → `GeminiEngine.remove_watermark()`
4. Convierte el resultado numpy → Pillow Image (RGB)
5. Si no se detecta watermark, retorna copia del original

**Manejo de errores:**
- Si `GeminiEngine` falla (excepción), retorna copia del original + log
- Si no se detecta watermark, retorna copia del original
- Soporta imágenes RGB, RGBA (convierte a RGB) y L (escala de grises → RGB)

### Modelo actualizado — `Backend/models.py`

```python
class ProcessingInstructions(BaseModel):
    ...
    remove_watermark: Optional[bool] = None
```

- `None`/`False` → no se aplica eliminación de watermark
- `True` → se aplica `WatermarkRemoverProcessor` antes de las transformaciones

### Pipeline actualizado — `Backend/main.py`

El endpoint ahora aplica el procesamiento en 3 pasos:

```python
# 1. Watermark removal (opcional, pre-procesamiento)
if order.instructions.remove_watermark:
    image_to_process = _watermark_remover.process(image_to_process, ...)

# 2. Transformaciones (resize, rotate, etc.)
processed_image = _processor.process(image_to_process, ...)

# 3. Formateo de salida
data, content_type = _formatter.format_output(processed_image, ...)
```

`WatermarkRemoverProcessor` se importa de forma lazy (try/except ImportError):
- Si la dependencia no está instalada, `_watermark_remover = None`
- Si el frontend solicita watermark removal y no está disponible, se loguea un warning
