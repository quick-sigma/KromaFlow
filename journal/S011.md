# S011 — Pipeline y Step ID

## Resumen

### Pipeline — Reglas de validación

1. Al menos un Step con `variant == PROCESSOR`
2. Exactamente un Step con `variant == OUTPUT_FORMATTER`
3. El OutputFormatter debe ser el **último** step en la lista

```python
Pipeline([proc_step, fmt_step])           # ✓ OK
Pipeline([proc_a, proc_b, fmt_step])      # ✓ OK (múltiples processors)
Pipeline([fmt_step])                       # ✗ Sin processor
Pipeline([proc_step])                      # ✗ Sin formatter
Pipeline([proc_step, fmt_step, fmt_step])  # ✗ Múltiples formatters
Pipeline([proc_step, fmt_step, proc])      # ✗ Formatter no es último
```

### Pipeline.execute()

```python
pipeline = Pipeline([ImageProcessorStep(), ImageOutputFormatterStep()])
data, ctype = pipeline.execute(
    image,
    configs={
        "img_proc": ImageProcessingConfig(grayscale=True),
        "img_fmt": OutputFormatConfig(format="jpeg", quality=90),
    },
)
```

- Si un step no tiene entrada en `configs`, se crea un default vacío (`config_schema()`)
- Los processors se encadenan: output de uno → input del siguiente
- El formatter final produce `(bytes, content_type)`

### pipeline_from_steps()

```python
pipeline = pipeline_from_steps(["img_proc", "img_fmt"])
```

Usa `_step_id_map` para resolver IDs a clases Step, las instancia y construye un `Pipeline`. Lanza `ValueError` si algún ID es desconocido.
