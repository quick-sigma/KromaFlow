# S002 — Configuración de tests

## Resumen
Se configuró el entorno de testing para Backend (FastAPI) y Frontend (React), y se creó un script unificado para ejecutar ambos.

## Detalle

### Backend (`Backend/`)
- Framework: **pytest** + **TestClient** de FastAPI (HTTPX)
- Dependencia instalada: `pytest`
- Archivos creados:
  - `tests/__init__.py`
  - `tests/test_main.py` — test del endpoint `GET /` que verifica status 200 y cuerpo `{"message": "Hello World"}`
- Ejecución: `source .venv/bin/activate && python -m pytest tests/ -v`

### Frontend (`frontend/`)
- Framework: **vitest** + **@testing-library/react** + **jsdom**
- Dependencias instaladas: `vitest`, `@testing-library/react`, `@testing-library/jest-dom`, `@testing-library/user-event`, `jsdom`
- Archivos creados:
  - `vite.config.ts` — se añadió bloque `test` con environment `jsdom`, setup file y `globals: true`
  - `src/setupTests.ts` — importa `@testing-library/jest-dom` para matchers como `toBeInTheDocument()`
  - `src/stores/counter.test.ts` — 3 tests unitarios del store Zustand (valor inicial, incremento, decremento)
  - `src/App.test.tsx` — 4 tests de componente (renderiza título, count inicial 0, incremento con click, decremento con click)
- Scripts añadidos a `package.json`:
  - `npm run test` — `vitest run` (una ejecución)
  - `npm run test:watch` — `vitest` (modo watch)

### Script unificado (`test.sh`)
- Ejecuta tests del backend y frontend secuencialmente
- Muestra resultados en consola con separadores claros
- Falla con `set -e` si algún test no pasa
- Ejecución: `./test.sh`

### Resultados
- Backend: **1 passed**
- Frontend: **2 test files, 7 tests passed**
