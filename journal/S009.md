# S009 — AVIF Output Formatter

## Resumen

Se creó `AVIFOutputFormatter`, una implementación concreta de `OutputFormatter` que codifica imágenes al formato AVIF (AV1 Image File Format), un formato moderno con mejor compresión que JPEG/WebP y soporte nativo de alpha.

Para integrarlo, se reemplazó el formatter único en `main.py` por un **registro de formatters** (`_formatters: dict[str, OutputFormatter]`) que enruta cada formato a su implementación correspondiente.

### AVIFOutputFormatter — `Backend/avif_output_formatter.py`

```python
class AVIFOutputFormatter(OutputFormatter):
    def format_output(self, image, output_format, quality=None) -> tuple[bytes, str]:
```

| Propiedad | Valor |
|---|---|
| Formatos soportados | `avif` |
| Content-Type | `image/avif` |
| Alpha | Nativo (RGBA funciona sin composición) |
| Calidad | 1–100, se pasa directo a Pillow |
| Conversiones | L (grayscale) → RGB para compatibilidad |

### Formatter Registry — `Backend/main.py`

```python
_formatters: dict[str, OutputFormatter] = {
    "png":   _image_formatter,   # ImageOutputFormatter
    "jpeg":  _image_formatter,   # ImageOutputFormatter
    "webp":  _image_formatter,   # ImageOutputFormatter
    "gif":   _image_formatter,   # ImageOutputFormatter
    "bmp":   _image_formatter,   # ImageOutputFormatter
    "tiff":  _image_formatter,   # ImageOutputFormatter
    "avif":  _avif_formatter,    # AVIFOutputFormatter ✨
}
```

El endpoint ahora resuelve el formatter así:

```python
formatter = _formatters.get(order.output_format)
data, content_type = formatter.format_output(processed_image, ...)
```

Esto permite agregar nuevos formatos simplemente creando una nueva clase `OutputFormatter` y registrándola en el diccionario.
